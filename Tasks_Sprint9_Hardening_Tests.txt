3. TASK 1 — REAL HIBP INTEGRATION (Server-Side)
════════════════════════════════════════════════════════════

BACKGROUND:
  Sprints 2 and 3 accepted a client-provided hibpCheckResult: boolean flag.
  This was marked as a stub. Sprint 9 replaces it with a real server-side
  k-anonymity HIBP API call.

WHAT IS k-ANONYMITY HIBP:
  The client never sends the full passphrase to the server.
  Instead: client hashes the passphrase with SHA-1, sends the first 5 chars (the prefix).
  The server sends those 5 chars to api.pwnedpasswords.com and receives back a list of
  SHA-1 hash suffixes + breach counts. Server checks if the client's full hash suffix
  appears in the list.

Since the server in DeathTrap does NOT receive the passphrase or its SHA-1 hash,
implement the HIBP check as follows:

Client sends (in RegisterCreatorRequest and ChangePassphraseRequest):
  hibpPrefix        String — first 5 chars of SHA-1(passphrase)  [already hex]
  hibpSuffix        String — remaining 35 chars of SHA-1(passphrase) [to match against list]
  hibpCheckResult   boolean — client's own result (keep as fallback)

Server behavior:
  1. Call HIBP API: GET https://api.pwnedpasswords.com/range/{hibpPrefix}
     - Set User-Agent header: "DeathTrap/1.0"
     - Timeout: 3 seconds
     - If API call fails (timeout, network error): fall back to trusting hibpCheckResult
       and log WARN: "HIBP API unavailable, trusting client flag"
  2. Parse the response: each line is "SUFFIX:COUNT"
     Search for a line where the suffix (uppercase) == hibpSuffix.toUpperCase()
  3. If found: throw AppException.of(ErrorCode.AUTH_PASSPHRASE_COMPROMISED)
  4. If not found: passphrase passes — proceed with registration/change.

Create: packages/common-crypto/src/main/java/in/deathtrap/common/crypto/HibpClient.java

@Component
@Slf4j
public class HibpClient {

  private static final String HIBP_API = "https://api.pwnedpasswords.com/range/";
  private static final int TIMEOUT_MS = 3000;

  /**
   * Checks if the passphrase appears in known data breaches using k-anonymity.
   * @param hibpPrefix first 5 hex chars of SHA-1(passphrase)
   * @param hibpSuffix remaining 35 hex chars of SHA-1(passphrase)
   * @param clientFlag client-side check result (used as fallback if API unavailable)
   * @throws AppException if passphrase is breached
   */
  public void checkPassphrase(String hibpPrefix, String hibpSuffix, boolean clientFlag) {
    // Validate inputs
    if (hibpPrefix == null || hibpPrefix.length() != 5) {
      throw AppException.of(ErrorCode.VALIDATION_FAILED);
    }
    if (hibpSuffix == null || hibpSuffix.length() != 35) {
      throw AppException.of(ErrorCode.VALIDATION_FAILED);
    }

    try {
      java.net.URL url = new java.net.URL(HIBP_API + hibpPrefix.toUpperCase());
      java.net.HttpURLConnection conn = (java.net.HttpURLConnection) url.openConnection();
      conn.setRequestMethod("GET");
      conn.setConnectTimeout(TIMEOUT_MS);
      conn.setReadTimeout(TIMEOUT_MS);
      conn.setRequestProperty("User-Agent", "DeathTrap/1.0");
      conn.setRequestProperty("Add-Padding", "true"); // HIBP padding for privacy

      int responseCode = conn.getResponseCode();
      if (responseCode != 200) {
        log.warn("HIBP API returned non-200: {} — trusting client flag={}", responseCode, clientFlag);
        checkClientFlag(clientFlag);
        return;
      }

      String suffixUpper = hibpSuffix.toUpperCase();
      try (java.io.BufferedReader reader = new java.io.BufferedReader(
               new java.io.InputStreamReader(conn.getInputStream()))) {
        String line;
        while ((line = reader.readLine()) != null) {
          String[] parts = line.split(":");
          if (parts.length >= 1 && parts[0].trim().equalsIgnoreCase(suffixUpper)) {
            // Found in breach list
            throw AppException.passphraseCompromised();
          }
        }
      }
      // Not found — passphrase is clean
    } catch (AppException e) {
      throw e; // re-throw AppException (breached)
    } catch (Exception e) {
      log.warn("HIBP API call failed: {} — trusting client flag={}", e.getMessage(), clientFlag);
      checkClientFlag(clientFlag);
    }
  }

  private void checkClientFlag(boolean clientFlag) {
    if (!clientFlag) {
      throw AppException.passphraseCompromised();
    }
  }
}

public abstract class IntegrationTestBase {

  @Container
  static final PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:15")
    .withDatabaseName("deathtrap_test")
    .withUsername("dt_test")
    .withPassword("dt_test_pw");

  @BeforeAll
  static void runMigrations() {
    Flyway.configure()
      .dataSource(postgres.getJdbcUrl(), postgres.getUsername(), postgres.getPassword())
      .locations("filesystem:../../migrations/sql")
      .load()
      .migrate();
  }

  protected DataSource dataSource() {
    HikariConfig config = new HikariConfig();
    config.setJdbcUrl(postgres.getJdbcUrl());
    config.setUsername(postgres.getUsername());
    config.setPassword(postgres.getPassword());
    return new HikariDataSource(config);
  }
}

6. TASK 4 — ARGON2id PARAMETER GUIDANCE DOCUMENT
════════════════════════════════════════════════════════════

The server does NOT run Argon2id (this is mobile-side only).
However, Sprint 9 must produce a reference document for the mobile team.

Create: ARGON2ID_PARAMETERS.md in the project root.

Content to include:
  Recommended parameters for DeathTrap (balance security vs mobile performance):
    Algorithm:   Argon2id
    Memory cost: 65536 KB (64 MB) — for mid-range Android/iOS
    Iterations:  3
    Parallelism: 2
    Output:      32 bytes (256 bits) — used as master key for HKDF

  Target benchmark: ≥ 500ms on the lowest-spec supported device.
  Devices to benchmark: Android Go (1GB RAM), iPhone SE 2nd gen.
  If benchmark shows < 500ms: increase iterations to 4 or memory to 128MB.
  If benchmark shows > 2s: decrease memory to 32MB as fallback.

  Cross-platform libraries:
    iOS:   CryptoKit does not support Argon2id. Use libsodium via swift-sodium or argon2-swift.
    Android: Argon2Kt library (https://github.com/lambdapioneer/argon2kt)

  Derivation chain:
    masterKey  = Argon2id(passphrase, salt, params) → 32 bytes
    lockerKey  = HKDF-SHA256(masterKey, salt="locker", info="AES256GCM") → 32 bytes
    kdfKey     = HKDF-SHA256(masterKey, salt="keypair", info="ECDH_P256") → 32 bytes
    [use kdfKey to deterministically derive ECDH keypair]

  NOTE: The salt (32 bytes CSPRNG) is generated on first registration and stored
  server-side in party_salts. It is NOT secret — only the passphrase provides entropy.

This document is for the mobile team — it should be written in plain English.

import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30s', target: 10 },   // ramp up to 10 VUs
    { duration: '60s', target: 10 },   // hold at 10
    { duration: '30s', target: 0 },    // ramp down
  ],
  thresholds: {
    http_req_failed:   ['rate<0.01'],  // error rate < 1%
    http_req_duration: ['p(95)<2000'], // 95th percentile < 2s
  },
};

const BASE = __ENV.BASE_URL || 'http://localhost:3000';

export default function () {
  // Test OTP send endpoint
  const resp = http.post(`${BASE}/auth/otp/send`,
    JSON.stringify({ mobile: '+919876543210', purpose: 'registration' }),
    { headers: { 'Content-Type': 'application/json' } }
  );
  check(resp, { 'otp send 200': (r) => r.status === 200 || r.status === 429 });
  sleep(1);
}
